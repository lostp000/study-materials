## 2. Protocol

- **Препроцессинг** 

  Scaling: StandardScaler - все признаки приведены к нулевому среднему и единичной дисперсии  
  Imputation: SimpleImputer с strategy='mean' - заполнение пропусков средними значениями  
  Encoding: Не требовалось (только числовые признаки)  
  PCA: Применялся только для визуализации (n_components=2)  

- **Поиск гиперпараметров**

  KMeans:   
    k от 2 до 20
  DBSCAN:  
    eps: [1.5, 2.0, 2.5, 3.0, 3.5]  
    min_samples: [3, 5, 10]  

- **Метрики**

  Silhouette Score (от -1 до 1, чем выше тем лучше)  
  Davies-Bouldin Score (чем ближе к 0 тем лучше)  
  Calinski-Harabasz Score (чем выше тем лучше)  
  Для DBSCAN: метрики считались только на non-noise точках после перенумерации кластеров 

- **Визуализация**

  PCA(2D) с стандартными параметрами sklearn  
  t-SNE не использовался

## 5. Analysis

### 5.1 Сравнение алгоритмов (важные наблюдения)
  - KMeans "ломается" когда кластеры не сферические или имеют разную плотность  
  - DBSCAN выигрывает при наличии шума и кластеров произвольной формы  
  - Наибольшее влияние: масштабирование признаков и наличие выбросов  

### 5.2 Устойчивость (обязательно для одного датасета)

**Выбран датасет:** Dataset 1 (S07-hw-dataset-01.csv)

**Метод проверки:** 
- 10 запусков KMeans с разными `random_state` (от 42 до 51)
- Расчет Adjusted Rand Index (ARI) между всеми парами запусков
- Дополнительно: нормализованная взаимная информация (NMI)

**Результаты:**
- Средний ARI: 0.985
- Стандартное отклонение ARI: 0.008
- Средний NMI: 0.972  
- Стандартное отклонение NMI: 0.010

**Интерпретация:**
- ARI > 0.95 указывает на высокую устойчивость
- Низкое стандартное отклонение подтверждает воспроизводимость
- Кластерная структура четкая и стабильная при разных инициализациях

**Вывод:** Алгоритм KMeans демонстрирует высокую устойчивость на данном датасете. 
Результаты кластеризации практически не зависят от начальной инициализации.

## 6. Conclusion

1. Масштабирование обязательно - без него алгоритмы работают некорректно  
2. Silhouette не всегда информативен - низкие значения не всегда означают плохую кластеризацию  
3. DBSCAN чувствителен к параметрам - требует тщательного подбора eps и min_samples  
4. Метод локтя полезен как эвристика - помогает выбрать k для KMeans  
5. Разные алгоритмы для разных структур - KMeans для сферических, DBSCAN для произвольных форм  
6. Визуализация помогает интерпретировать - PCA показывает реальную структуру данных  
7. Несколько метрик лучше одной - нужна комплексная оценка качества  
8. Проверка устойчивости важна - особенно для алгоритмов с случайной инициализацией
